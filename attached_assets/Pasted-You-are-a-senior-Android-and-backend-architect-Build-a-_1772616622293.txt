You are a senior Android and backend architect.

Build a complete production-ready Android application.

PROJECT NAME
Smart GPS + QR Based Campus Attendance System

TECH STACK
Android App:

* Language: Java
* Architecture: MVVM
* Android Studio
* Retrofit (API calls)
* ZXing (QR scanner)
* Google Location Services
* Material UI

Backend:

* Node.js + Express
* MongoDB
* JWT Authentication

IMPORTANT SECURITY RULE
The app must NOT allow users to access the system unless they are physically inside the campus GPS range.

---

APP ENTRY SECURITY (VERY IMPORTANT)

When the app opens:

1. The system immediately checks GPS location.
2. The campus coordinates are fixed.

Example:

Campus Latitude: 19.0760
Campus Longitude: 72.8777
Allowed Radius: 300 meters

3. If the user is OUTSIDE the campus radius:

* App should show:

"Access Denied
You are outside the campus area."

* Login screen must NOT open.

4. If the user is INSIDE the campus radius:

* Allow login screen.

TEACHER FEATURES

Teacher Dashboard

* See assigned classes
* View attendance history

Generate Attendance Session

Teacher clicks:

"Start Attendance"

System will:

1. Capture teacher GPS location
2. Generate a unique QR token
3. Create a QR code
4. QR expires in 60 seconds

Students must scan within that time.

---

STUDENT FEATURES

Student Dashboard

* Attendance percentage
* Subject attendance list

Mark Attendance

Steps:

1. Student opens scanner
2. Scan QR code
3. App captures GPS location
4. App sends:

studentId
sessionToken
latitude
longitude

Backend validation:

Check 1:
QR session valid?

Check 2:
Session expired?

Check 3:
Student within 100 meters of teacher?

Check 4:
Student already marked attendance?

If valid:

Mark attendance.

If invalid:

Show error message.
